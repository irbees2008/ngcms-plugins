# ✅ Универсальные Telegram уведомления для NGCMS - ГОТОВО К РАБОТЕ!

## 🎯 Что это?

Плагин для отправки уведомлений администратору в Telegram о различных событиях на сайте:

- 💬 **jChat** - новые сообщения в чате
- 📧 **Feedback** - обратная связь через формы
- ✉️ **PM** - личные сообщения
- ⚠️ **Complaints** - жалобы на контент
- 💭 **Comments** - новые комментарии (опционально)

## ⚡ Быстрый старт

### 1. Создайте Telegram бота

1. Откройте [@BotFather](https://t.me/BotFather)
2. Отправьте `/newbot`
3. Придумайте имя (Example Bot)
4. Придумайте username (example_bot)
5. Получите токен вида: `123456789:ABCdefGHIjklMNOpqrsTUVwxyz`

### 2. Узнайте свой Chat ID

1. Откройте [@userinfobot](https://t.me/userinfobot)
2. Нажмите START
3. Получите ID вида: `123456789`

### 3. Настройте плагин

1. Админ-панель → Плагины → jChat Telegram Уведомления
2. **Включить уведомления**: Да
3. **Bot Token**: вставьте токен из шага 1
4. **Chat ID**: вставьте ID из шага 2
5. Выберите типы уведомлений (галочки):
   - ✅ jChat (уже работает автоматически)
   - ✅ Обратная связь (требует интеграции)
   - ✅ Личные сообщения (требует интеграции)
   - ✅ Жалобы (требует интеграции)
6. Сохраните

### 4. Добавьте интеграцию (опционально)

См. файл [INTEGRATION_GUIDE.md](INTEGRATION_GUIDE.md) для добавления уведомлений в другие плагины.

## 🛠️ Технические особенности

### Безопасность

- ✅ Санитизация всех данных через `sanitize()`
- ✅ SSL верификация HTTPS запросов
- ✅ Проверка HTTP кодов ответа
- ✅ Try-catch обработка ошибок

### Логирование

Все операции логируются в `engine/data/logs/plugins.log`:

- INFO - успешные отправки
- WARNING - пустые сообщения, отключенные уведомления
- ERROR - ошибки при отправке
- DEBUG - детальная информация

### API

```php
// Отправка универсального уведомления
ngcms_tg_notify('TYPE', [
    'title'    => 'Заголовок',
    'author'   => 'Автор',
    'text'     => 'Текст',
    'url'      => 'https://site.ru',
    'datetime' => date('Y-m-d H:i:s'),
]);

// TYPE: jchat, feedback, pm, complain, comment
```

## 🔧 Управление фильтрами:

### "Только гости" - кому отправлять

- **ВКЛ** → Только от незарегистрированных пользователей
- **ВЫКЛ** → От всех (гости + зарегистрированные)

### "Только первое сообщение" - как часто

- **ВКЛ** → Одно уведомление за сессию от посетителя
- **ВЫКЛ** → Каждое сообщение (может быть много!)

### "Антифлуд" - интервал

- **0** → Без ограничений (не рекомендуется)
- **20-30** → Оптимально
- **60+** → Редкие уведомления

## 🛠️ Техническая информация:

### Как работает:

1. Посетитель пишет в jChat
2. jChat сохраняет в БД
3. Вызывается `jchat_tgnotify_send()`
4. Проверяются фильтры
5. Отправка через Telegram Bot API
6. Вы получаете уведомление

### API используемые:

- `extra_get_param()` - чтение настроек NGCMS
- `extra_set_param()` - сохранение настроек
- Telegram Bot API `sendMessage`
- PHP cURL или file_get_contents

### Требования:

- PHP 7.0+
- jChat установлен
- Интернет-соединение к api.telegram.org

## ❓ Часто задаваемые вопросы:

**Q: Можно ли отправлять в несколько чатов?**
A: Нет, только в один Chat ID. Но можно создать группу и добавить туда бота.
**Q: Как отключить уведомления временно?**
A: Настройки → Включить уведомления → Нет
**Q: Можно изменить текст уведомления?**
A: Да, отредактируйте функцию `jchat_tgnotify_send()` в файле `jchat_tgnotify.php`
**Q: Работает ли с несколькими сайтами?**
A: Да, создайте отдельного бота для каждого сайта.
