## Применённые функции ng-helpers

### 1. cache_get / cache_put (Категория: Cache) - v0.2.0

- **Назначение:** Замена устаревших cacheRetrieveFile/cacheStoreFile на современный API кеширования
- **Использование:**

  ```php
  // Старый код:
  $cacheFileName = md5('ads_pro' . $blockID . '.' . $blockIndexNum . '.' . $blockInfo['type']) . '.txt';
  $cacheData = cacheRetrieveFile($cacheFileName, 30000, 'ads_pro');
  cacheStoreFile($cacheFileName, $description, 'ads_pro');

  // Новый код (v0.2.2):
  $cacheKey = 'ads_pro:' . $blockID . ':' . $blockIndexNum . ':' . $blockInfo['type'];
  $cacheData = cache_get($cacheKey);

  if ($cacheData !== null) {
      logger('ads_pro', 'Cache HIT for block: id=' . $blockIndexNum, 'debug');
  } else {
      logger('ads_pro', 'Cache MISS for block: id=' . $blockIndexNum, 'debug');
  }

  // Настраиваемая длительность кеша
  $cacheDuration = clamp(intval(pluginGetVariable('ads_pro', 'cache_duration', 30000)), 300, 86400);
  cache_put($cacheKey, $description, $cacheDuration);
  ```

- **Преимущества:**
  - Поддержка множественных драйверов (файлы, Redis, Memcached)
  - Чистые ключи вместо MD5 хешей
  - Строгая проверка null вместо false
  - Настраиваемый TTL от 5 минут до 24 часов
  - Логирование попаданий/промахов кеша
  - Лучшая читаемость кода

### 2. sanitize (Категория: Security) - v0.2.0

- **Назначение:** Очистка HTML контента и описаний рекламных блоков
- **Использование:**

  ```php
  // Для HTML блоков (type=0)
  if ($blockInfo['type'] == 0) {
      $description = sanitize($row['ads_blok'], 'html');
  }

  // Для TEXT блоков (type=2)
  else {
      $description = nl2br(sanitize($row['ads_blok'], 'string'));
  }

  // Для восстановленных блоков
  'description' => $row['description'] ? sanitize($row['description'], 'string') : 'Восстановленный блок #' . $dbId,
  ```

- **Преимущества:**
  - Безопасная обработка HTML в рекламных блоках (тип 0)
  - Очистка от вредоносного кода
  - Защита от XSS атак через рекламные блоки
  - Валидация восстановленных данных при синхронизации
  - Раздельная обработка HTML и TEXT типов

### 3. logger (Категория: Debugging) - v0.2.0+, улучшено v0.2.2

- **Назначение:** Комплексное логирование операций с рекламными блоками
- **Использование v0.2.2:**

  ```php
  // Кеш операции
  logger('ads_pro', 'Cache HIT for block: id=' . $blockIndexNum . ', block=' . $blockID, 'debug');
  logger('ads_pro', 'Cache MISS for block: id=' . $blockIndexNum . ', block=' . $blockID, 'debug');

  // Рендеринг блоков
  logger('ads_pro', 'Rendered and cached block: id=' . $blockIndexNum . ', size=' . strlen($description) . ' bytes', 'info');

  // PHP блоки с производительностью
  logger('ads_pro', 'Executing PHP block: id=' . $blockIndexNum . ', block=' . $blockID, 'info');
  logger('ads_pro', 'PHP block executed: id=' . $blockIndexNum . ', time=' . round($result['time'] * 1000, 2) . 'ms, output=' . strlen($out2) . ' bytes', 'info');

  // Синхронизация с уровнями важности
  logger('ads_pro', 'Starting database synchronization', 'debug');
  logger('ads_pro', 'Sync: removing ' . count($missingInDb) . ' orphaned blocks from config, IDs=' . implode(',', $missingInDb), 'warning');
  logger('ads_pro', 'Sync: restoring ' . count($missingInConfig) . ' blocks from database, IDs=' . implode(',', $missingInConfig), 'warning');
  logger('ads_pro', 'Restored block ' . $dbId . ' from original config', 'info');
  logger('ads_pro', 'Database synchronization completed, config saved', 'info');
  ```

- **Уровни важности:**
  - `debug` - детальная информация о кеше и синхронизации
  - `info` - успешные операции, рендеринг блоков
  - `warning` - восстановление/удаление блоков
  - `error` - критические ошибки (будущее расширение)
- **Преимущества:**
  - Полное отслеживание жизненного цикла блоков
  - Мониторинг попаданий/промахов кеша
  - Контроль производительности PHP блоков
  - Аудит синхронизации БД при переносе сайта
  - Анализ размера рендеренного контента

### 4. benchmark (Категория: Performance) - НОВОЕ v0.2.2

- **Назначение:** Мониторинг производительности выполнения PHP блоков
- **Использование:**

  ```php
  // Замер времени выполнения PHP кода
  $result = benchmark(function() use ($description, &$out2) {
      ob_start();
      @eval($description);
      $out2 = ob_get_contents();
      ob_end_clean();
  });

  logger('ads_pro', 'PHP block executed: id=' . $blockIndexNum . ', time=' . round($result['time'] * 1000, 2) . 'ms, output=' . strlen($out2) . ' bytes', 'info');
  ```

- **Возвращаемые данные:**
  - `time` - время выполнения в секундах (float)
  - `memory` - использованная память в байтах
  - `peak_memory` - пиковое использование памяти
- **Преимущества:**
  - Выявление медленных PHP блоков
  - Оптимизация производительности рекламы
  - Мониторинг использования памяти
  - Выявление блоков требующих кеширования
  - Автоматическое логирование результатов

### 5. clamp (Категория: Validation) - НОВОЕ v0.2.2

- **Назначение:** Валидация и ограничение числовых параметров
- **Использование:**

  ```php
  // Ограничение длительности кеша от 5 минут до 24 часов
  $cacheDuration = clamp(intval(pluginGetVariable('ads_pro', 'cache_duration', 30000)), 300, 86400);

  // Валидация типа блока (0=HTML, 1=PHP, 2=TEXT)
  'type' => clamp(intval($row['type']), 0, 2),

  // Валидация состояния (0=off, 1=on, 2=scheduled)
  'state' => clamp(intval($row['state']), 0, 2),
  ```

- **Преимущества:**
  - Защита от некорректных значений в БД
  - Предотвращение переполнения кеша
  - Валидация восстановленных данных
  - Безопасность при миграции сайта
  - Читаемость кода вместо if/else

### 6. array_get / array_only (Категория: Arrays) - НОВОЕ v0.2.2

- **Назначение:** Безопасная работа с конфигурацией блоков
- **Подготовлено для использования:**

  ```php
  // Безопасное получение значений из конфигурации
  $blockName = array_get($blockConfig, 'name', 'Unnamed Block');
  $blockType = array_get($blockConfig, 'type', 0);

  // Фильтрация только нужных полей при восстановлении
  $cleanConfig = array_only($row, ['id', 'description', 'type', 'state', 'location']);
  ```

- **Преимущества:**
  - Защита от отсутствующих ключей
  - Значения по умолчанию
  - Чистая работа с массивами
  - Упрощение валидации

### 7. is_ajax (Категория: HTTP) - НОВОЕ v0.2.2

- **Назначение:** Определение AJAX запросов для динамической загрузки блоков
- **Подготовлено для будущего использования:**
  ```php
  // Динамическая загрузка рекламных блоков через AJAX
  if (is_ajax()) {
      // Отдать только JSON с блоком, без шаблона
      header('Content-Type: application/json');
      echo json_encode(['block' => $description, 'id' => $blockIndexNum]);
      exit;
  }
  ```
- **Преимущества:**
  - Поддержка асинхронной загрузки рекламы
  - Снижение времени первой отрисовки страницы
  - Отложенная загрузка тяжелых блоков
  - API для внешних систем

```bash
# Просмотр логов кеша
```

### 5. Оптимизация производительности

- Используйте Redis/Memcached для высоконагруженных сайтов
- Увеличьте TTL (30000 сек) для редко меняющихся блоков

## Типы рекламных блоков v0.2.2

### Тип 0: HTML контент

- **Кешируется:** Да (настраиваемый TTL, по умолчанию 30000 секунд)
- **Обработка:** sanitize($content, 'html') для безопасности
- **Логирование:** Cache HIT/MISS, размер контента
- **Использование:** Статические баннеры, Google AdSense, HTML виджеты
- Применяется sanitize для безопасности
- **Рекомендуется** для Google AdSense, баннеров, HTML кода

### Тип 1: PHP код

- **Кешируется:** НЕТ (динамическое выполнение)
- **Выполняется:** eval() — потенциально опасно
- **Мониторинг:** benchmark() + логирование времени и памяти
- **Логирование:** Каждый запуск с деталями производительности
- **Использование:** Динамические блоки, сложная логика
- **Пример лога:** `PHP block executed: id=12, time=45.23ms, output=1536 bytes`
- **⚠️ Не рекомендуется** для production без строгого контроля

### Тип 2: Текст

- **Кешируется:** Да (настраиваемый TTL, по умолчанию 30000 секунд)
- **Обработка:** sanitize($content, 'string') + nl2br()
- **Логирование:** Cache HIT/MISS, размер контента
- **Использование:** Простые текстовые объявления, информация
- **Безопасность:** HTML экранируется, переводы строк → `<br>`
- **✅ Рекомендуется** для простых текстовых блоков без HTML

## Частые сценарии использования

### 1. Google AdSense (HTML блок, тип 0)

```html
<script
  async
  src="https://pagead2.googlesyndication.com/pagead/js/adsbygoogle.js"></script>
<ins
  class="adsbygoogle"
  style="display:block"
  data-ad-client="ca-pub-XXXXX"
  data-ad-slot="XXXXX"></ins>

  (adsbygoogle = window.adsbygoogle || []).push({});
</script>
```

- **Тип:** 0 (HTML)
- **Кешируется:** Да, настраиваемый TTL (рекомендуется 43200 = 12 часов)
- **Обработка:** sanitize для безопасности
- **Логирование:** Cache HIT/MISS + размер контента
- **Показывается:** По условиям (главная, категории, и т.д.)

### 2. Динамический PHP блок (тип 1) - С ОСТОРОЖНОСТЬЮ

```php
// Показать текущую дату и время
echo '<div class="ad-date">' . date('d.m.Y H:i') . '</div>';

// Или динамический счётчик
global $mysql;
$count = $mysql->result("SELECT COUNT(*) FROM " . prefix . "_news");
echo "Всего новостей: $count";
```

- **Тип:** 1 (PHP)
- **Кешируется:** НЕТ
- **Мониторинг:** benchmark() — автоматический замер времени и памяти
- **Логирование:** Каждое выполнение с деталями
- **⚠️ Риски:** eval() — используйте только проверенный код
- **Пример лога:** `PHP block executed: id=12, time=3.45ms, output=156 bytes`

### 3. Текстовый блок (тип 2)

- **Использование:** Простые объявления, информационные блоки

## Известные проблемы и решения

### 1. Eval PHP кода (тип 1)

- **Проблема:** Небезопасное выполнение произвольного кода
- **Решение v0.2.2:**
  - Полное логирование каждого выполнения
  - Benchmark для контроля производительности
  - Рекомендация: минимизировать использование, проверять код
- **Мониторинг:** Смотреть логи на медленные блоки (> 100ms)

### 2. Синхронизация при переносе сайта

- **Проблема:** Расхождения между БД и конфигом
- **Решение:** Автоматическая синхронизация + логирование

### 3. Множественный показ

- **Проблема:** Может показывать много блоков одновременно
- **Решение:** Настройка `multidisplay_mode` (1=первый, 2=случайный)
